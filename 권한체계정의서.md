# 권한 체계 정의서 (v1.0)

본 문서는 역할(Role)별 접근 권한과 게시글 가시성 규칙을 정의합니다.

## 1. 역할 정의 (Role Definition)

| 역할 ID     | 역할명 | 설명                                                          |
| :---------- | :----- | :------------------------------------------------------------ |
| **admin**   | 관리자 | 시스템의 모든 권한을 가짐 (김관리자 등)                       |
| **manager** | 담당자 | 본인 업무와 공지사항 위주의 접근 권한 (박담당자, 정개발자 등) |

## 2. 게시글 가시성 규칙 (Visibility Rules)

| 구분            | 관리자 (Admin)             | 담당자 (Manager)                     |
| :-------------- | :------------------------- | :----------------------------------- |
| **일반 게시글** | 모든 게시글 조회 가능      | **본인이 작성한 글만** 조회 가능     |
| **공지 게시글** | 모든 공지글 조회/관리 가능 | **모든 공지글** 조회 가능            |
| **비밀 게시글** | 모든 비밀글 조회 가능      | **본인이 작성한 비밀글만** 조회 가능 |

## 3. 기능 권한 매트릭스

| 기능                    | 관리자    | 담당자      | 비고                            |
| :---------------------- | :-------- | :---------- | :------------------------------ |
| **게시글 작성**         | O         | O           |                                 |
| **상태 변경**           | O         | X           | 담당자는 읽기 전용 배지로 노출  |
| **내부 메모 작성/조회** | O         | X           | 담당자에게는 UI가 노출되지 않음 |
| **작업공수 입력/조회**  | O         | X           | 담당자에게는 UI가 노출되지 않음 |
| **댓글 삭제**           | 모든 댓글 | 본인 댓글만 |                                 |

## 4. 접근 제어 로직 (Pseudocode)

```typescript
// 게시판 목록 필터링 로직 예시
const getAccessibleBoards = (user, allBoards) => {
  if (user.role === "admin") return allBoards;

  return allBoards.filter(
    (board) => board.authorId === user.id || board.status === "notice"
  );
};
```
